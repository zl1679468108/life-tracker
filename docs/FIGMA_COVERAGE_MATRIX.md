## 覆盖原则

- 不新增一级 Tab，底部固定为 `首页 / 工作台 / 消息 / 我的`。
- 工作台直接平铺所有业务模块，不设“更多工具”。
- 我的页外层直接平铺账号、偏好、数据与支持入口。
- 默认页面形态为 `列表 / 新增 / 编辑`；无特殊要求不输出独立详情页。
- 图标区域必须使用线性图标；没有合适线性图标时使用模块名称中的单字作为文字图标。
- 深色和浅色模式都必须有主题依据；低频模块复用同一套列表/表单/设置行模板。

## Figma 页面补齐建议

受 Starter 计划 3 页限制影响，不再新增第 4 个 Figma Page；应在现有 `02 Screens & Interactions` 页面内追加以下区块：

1. `Coverage Matrix`
   - 一张横向矩阵板，列出所有 PRD 模块、入口、页面形态、状态。
2. `Utility Screens`
   - 分类、位置、模板、借用、共享、反馈、账号、密码、主题、语言。
3. `Data & Reminder Screens`
   - 数据统计、通知中心、数据管理、资产总览、桌面小组件。
4. `Context Capabilities`
   - AI 物品识别、模板套用、分享、借用上下文能力，明确不作为独立一级入口。

## 审美和交互约束

- 产品类型: 生活效率工具 + 轻量操作台。
- 视觉方向: 深色操作台为主、浅色模式等价，不做单色大面积铺满。
- 组件密度: 移动端可单手扫描，列表行为主，低频功能不做夸张卡片宫格。
- 触控目标: 所有行、按钮、图标按钮不小于 `44 x 44`。
- 字体层级: 页面标题 `22/30/700`，区块标题 `16/24/700`，正文 `13/20/400`。
- 图标规范: 工作台和我的入口使用线性图标；无法准确表达时使用单字文字图标。
- 状态覆盖: loading、empty、error、selected、disabled、danger confirmation 都需要在 Handoff 中明确。

## Handoff 门禁

补齐稿确认前，第三阶段编码应暂停在当前进度，不继续扩大实现范围。
