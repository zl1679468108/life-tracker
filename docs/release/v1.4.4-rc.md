# LifeTracker v1.4.4 — 发布候选 (RC)

> **版本**: v1.4.4-rc  
> **发布日期**: 2026-07-04  
> **目标平台**: Web (PWA) + Android  
> **技术栈**: Expo React Native SDK 54 + NestJS v11 + Supabase PostgreSQL

---

## 1. 版本概要

v1.4.4 聚焦**消息模块重设计、首页可视化增强、UI/UX 打磨**。不新增独立业务入口，将现有主干增强转为可交付版本。

| 维度 | 说明 |
|---|---|
| 新增文件 | `life_conversation_reads` 表、`useDebounce` hook、QA 冒烟脚本 |
| 修改文件 | 前端 ~30 个、后端 ~10 个、文档 5 个 |
| 删除文件 | 工作台搜索入口、系统消息对话入口、RLS 策略定义、零散未跟踪文件 (h5/, overview.md) |
| 风险等级 | **低** — 无 Schema 破坏性变更（仅新增字段/表），无后端路由结构变更 |

## 5. 残留风险

| 风险 | 等级 | 说明 | 缓解措施 |
|---|---|---|---|
| Web 端提醒依赖前端 `setTimeout` | **低** | `scheduleTodoReminder` 中 Web 端使用 `setTimeout` 实现定时通知；浏览器关闭后定时器丢失 | 提醒数据已写入 `life_reminder_logs`，下次打开时通知中心展示；不影响功能完整性 |
| 开发生产共用数据库 | **低** | 当前 Supabase 实例同时用于开发和生产 | 表结构变更不影响现有数据；所有迁移语句使用 `IF NOT EXISTS` 保证幂等 |
| 无 Service Worker 推送 | **低** | SW 仅做离线缓存，不支持推送通知 | 所有通知通过 WebSocket + 本地 Store 展示 |
| 双账号冒烟依赖环境变量 | **中** | `QA_USER_A/B_EMAIL` 和密码需在 `.env` 中配置 | 阻塞条件和复跑命令已记录在 `docs/qa/` |

## 6. 回滚提示

### 6.1 数据库回滚

```sql
-- 可选回滚：删除新增表
DROP TABLE IF EXISTS life_conversation_reads;

-- 可选回滚：删除新增列
ALTER TABLE life_reminder_logs DROP COLUMN IF EXISTS reminder_key;

-- 重新启用 RLS（不推荐，因 service_role 绕过问题）
-- 需要重新创建所有 POLICY，详见 git history 中的旧 database-init.sql
```

### 6.2 前端回滚

```bash
git checkout <pre-v1.4.4-tag> -- frontend/

```

### 6.3 后端回滚

```bash
git checkout <pre-v1.4.4-tag> -- backend/

```
