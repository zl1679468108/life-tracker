> 版本: v1.3.1

> 范围: 首页、工作台、物品列表、待办列表、消息、我的、底部导航及其核心跳转规则

- `ui-ux-pro-max`: 审查信息架构、触控目标、导航一致性、状态反馈和移动端可读性。

## 1. 原型目标

本次原型以“个人生活操作台”为设计方向，解决旧页面视觉层级弱、功能入口重复、页面职责交叉的问题。

- 首页承载今日概览和最高频操作，不展示完整管理列表。
- 工作台承载全部功能入口，不展示业务数据和表单。
- 物品、待办、统计、通知等复杂任务进入独立二级页完成。
- 底部导航固定为四个一级区域：首页、工作台、消息、我的。

## 2. 设计系统

| Token | 值 | 用途 |
|---|---|---|
| 页面背景 | `#0F1724` | 首页、工作台主背景 |
| 顶栏/卡片 | `#000000` | Header、统计容器、快捷卡、待办行、Tab Bar |
| 主文字 | `#FFFFFF` | 标题、数字、主要标签 |
| 次文字 | `#9CA3AF` | 描述、未选中导航 |
| 橙色 | `#FF8754` | 首页选中态、物品、添加物品 |
| 紫色 | `#8B68F5` | 待办、统计、通知、工具入口 |
| 绿色 | `#32D296` | 完成、添加待办、数据导出 |
| 黄色 | `#FBB329` | 借用登记 |

组件规则：

- 图标统一使用 `MaterialCommunityIcons`。
- 首页大按钮图标为黑色 glyph + 彩色圆角方块。
- 大卡片圆角 24-30px；图标容器圆角 20px；底部 Tab 无浮层卡片。
- 字号层级：页面标题 24-30px，卡片标题 21-22px，描述 18px 左右，导航标签 13px。

## 3. 页面原型

### 3.1 首页 `/`

布局从上到下：

1. 顶部 Header
   - 左侧显示时间问候：早上好/下午好/晚上好。
   - 副标题固定为“今天也要加油哦”。
   - 右侧两个圆形按钮：搜索、通知。
   - 通知有未读时显示红点并保留轻微抖动。

2. 统计总览
   - 黑色圆角容器内放 3 张统计卡。
   - 物品卡：橙色，显示物品总数，点击 `/item/list`。
   - 待办卡：紫色，显示未完成数，点击 `/todo/list`。
   - 完成卡：绿色，显示已完成数，不跳转。

3. 快捷操作
   - 2 列 4 卡片。
   - 添加物品：点击 `/item/create`。
   - 添加待办：点击 `/todo/create`。
   - 数据统计：点击 `/settings/stats`。
   - 通知中心：点击 `/settings/notifications`。
   - 不允许在该区域继续新增低频入口。

4. 最近待办
   - 标题左侧“最近待办”，右侧“查看全部”。
   - 列表最多 3 条，未完成优先。
   - 每行包含复选框、标题、描述、优先级标签。
   - 点击行进入 `/todo/[id]`；点击复选框仅切换完成状态。

5. 即将过期提醒
   - 有数据时显示，复用现有 `ExpiryWarning`。
   - 点击物品进入 `/item/[id]`。

异常状态：

- 首页数据加载失败时由 `SafeScreen` 显示错误 Snackbar。
- 无待办时隐藏“最近待办”区域，不显示空卡片。
- 无过期物品时隐藏过期提醒。

### 3.2 工作台 `/workbench`

布局：

- 页面标题：“功能入口”。
- 3 列网格，入口固定 11 个。
- 每个入口只包含彩色图标块和功能名称，不展示描述、副标题、统计数字或列表。

入口路由：

| 入口 | 路由 |
|---|---|
| 物品管理 | `/item/list` |
| 待办管理 | `/todo/list` |
| 借用登记 | `/settings/borrowing-create` |
| 模板使用 | `/settings/templates` |
| 分类管理 | `/settings/category-manage` |
| 位置管理 | `/settings/location-manage` |
| 日历视图 | `/settings/calendar` |
| 数据导出 | `/settings/data` |
| 共享管理 | `/settings/shares` |
| 统计概览 | `/settings/stats` |
| 通知中心 | `/settings/notifications` |

异常状态：

- 工作台不依赖远程数据，无加载态。
- 路由目标页负责权限、加载、错误和空态。

### 3.3 底部导航

四个 Tab：

| Tab | 图标 | 路由 | 规则 |
|---|---|---|---|
| 首页 | `home` | `/(tabs)` | 默认入口 |
| 工作台 | `grid` | `/(tabs)/workbench` | 功能入口 |
| 消息 | `message-text-outline` | `/(tabs)/messages` | 有未读时显示角标 |
| 我的 | `account` | `/(tabs)/settings` | 个人资料和系统设置 |

交互规则：

- 当前选中为橙色，未选中为灰色。
- Tab Bar 固定黑色背景。
- 消息未读角标最多显示 `99+`。

### 3.4 物品管理 `/item/list`

布局：

- 顶部自定义标题区：`物品管理` + 说明文案 + 总数胶囊。
- 右侧工具按钮：批量选择、排序。
- 搜索框：深色表面、主色聚焦边框、清空按钮。
- 分类筛选：圆形 chips，选中态橙色填充。
- 列表卡片：图标/缩略图、名称、描述、位置标签、分类标签、进入箭头。
- 批量模式：卡片显示复选框，底部浮动批量栏显示已选数量、全选、删除。
- 排序面板：底部 sheet，选项为添加时间、名称、分类。

交互：

- 点击卡片进入 `/item/[id]`。
- 左滑删除触发确认弹窗。
- FAB 进入 `/item/create`。
- 空状态显示添加引导。

### 3.5 待办管理 `/todo/list`

布局：

- 顶部自定义标题区：`待办管理` + 说明文案 + 待完成数量胶囊。
- 分段筛选：全部、待完成、已完成。
- 右侧工具按钮：拖拽排序、排序。
- 待办卡片：复选框、标题、描述、优先级标签、截止日期。
- 排序面板：底部 sheet，选项为添加时间、优先级、名称。

交互：

- 点击卡片进入 `/todo/[id]`。
- 点击复选框切换完成状态，不进入详情。
- 开启拖拽模式后长按拖拽排序。
- 左滑删除触发确认弹窗。
- FAB 进入 `/todo/create`。

### 3.6 消息 `/messages`

布局：

- 顶部标题：`消息` + 说明文案 + 对话数量胶囊。
- 常联系好友横向列表。
- 对话列表：头像、名称、最后消息摘要、时间、未读角标。
- FAB：新建对话。
- 新建对话底部面板：搜索用户 / 好友列表两个模式。

交互：

- 点击对话进入 `/message/[id]`。
- 搜索用户 300ms 防抖。
- 选择用户后按钮变为可点击，创建成功进入对话详情。
- 空对话状态显示发起对话引导。

### 3.7 我的 `/settings`

布局：

- 顶部标题：`我的` + 说明文案。
- 个人资料卡：头像、用户名、邮箱、进入箭头。
- 同步概览：同步状态、上次同步。
- 设置分组：数据同步/数据管理、主题/语言/修改密码、版本/反馈。

交互：

- 点击资料卡进入 `/settings/account`。
- 点击数据同步执行 `syncAll`，同步中显示 loading。
- 其他设置项进入对应二级页。

## 4. 页面层级与跳转

```text
首页
├── 搜索按钮 -> GlobalSearch 弹层
├── 通知按钮 -> /settings/notifications
├── 物品统计 -> /item/list
├── 待办统计 -> /todo/list
├── 添加物品 -> /item/create
├── 添加待办 -> /todo/create
├── 数据统计 -> /settings/stats
├── 最近待办行 -> /todo/[id]
└── 查看全部 -> /todo/list

工作台
└── 功能入口 -> 对应二级页面
```

## 5. 权限与数据规则

- 所有业务页面要求登录态；未登录由路由守卫进入登录页。
- 首页只读取当前用户有权限访问的物品、待办、通知。
- 共享资源只允许所有者和参与者访问。
- 系统预设分类/位置只读；用户自定义分类/位置可编辑。
- 创建/编辑类页面提交北京时间字符串；后端入库统一转 UTC ISO。

## 6. 开发验收

- 首页与工作台必须匹配本原型的深色背景、卡片结构、网格比例、Tab Bar 颜色。
- 首页不出现完整物品列表、完整待办列表、工作台入口全集。
- 工作台不出现数据列表、统计卡、表单、冗余说明。
- `npx tsc --noEmit` 通过。
- Web 端刷新后可恢复当前路由。
