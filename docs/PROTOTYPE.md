# LifeTracker 原型方案

> **版本**: v1.4.4  
> **日期**: 2026-07-03  
> **范围**: 首页、工作台、消息、我的、物品管理、待办管理、分类/位置、借用、模板、日历、统计、通知、数据管理、资产、桌面快捷入口及全局搜索

## 1. 概述

### 1.1 产品定位

LifeTracker 是一款个人生活管理工具，帮助用户追踪物品位置、管理待办事项，通过分类/位置体系和共享协作实现生活条理化。

### 1.2 技术底座

| 层 | 技术 | 说明 |
|---|---|---|
| 前端框架 | Expo React Native SDK 54 + expo-router v6 | 跨平台（Web + Android） |
| 状态管理 | Zustand v5 | 按领域拆分独立 store |
| UI 组件 | react-native-paper v5 + 自定义组件库 | 统一深浅色主题 |
| 后端 | NestJS v11 | REST API |
| 数据库 | Supabase PostgreSQL + RLS | Auth + Storage + PG |
| 实时通信 | WebSocket (Socket.io) | 消息推送、提醒通知 |

### 1.3 运行环境

| 服务 | 地址 |
|---|---|
| 前端开发 | `http://localhost:3021` |
| 后端开发 | `http://localhost:3020` |
| 前端构建产物 | `frontend/dist/` |
| 后端构建产物 | `backend/dist/` |

### 1.4 导航架构

底部 4 Tab 固定（首页 / 工作台 / 消息 / 我的），工作台为功能入口网格，所有业务功能通过 Stack 导航从工作台或上下文入口进入二级页。

## 2. 页面原型

### 2.1 首页 `/(tabs)`

**职责**: 今日总览 + 最高频操作，不展示完整业务列表。

布局（从上到下）:

1. **顶部 Header**
   - 左侧时间问候："早上好/下午好/晚上好"
   - 副标题："今天也要加油哦"
   - 右侧：搜索图标（进入全局搜索）、通知铃铛（未读红点 + 抖动动画，点击进入通知中心）

2. **"今天要看"行动区**
   - 展示逾期待办、今日待办、即将过期物品
   - 每项可点击跳转到对应编辑页
   - 无数据时不展示该区域

3. **数据统计卡片（3 张）**
   - 物品卡（橙色）：总数 → `/item/list`
   - 待办卡（紫色）：未完成数 → `/todo/list`
   - 完成卡（绿色）：已完成数（不可点击）

4. **快捷操作（2 个主按钮）**
   - 添加物品 → `/item/create`
   - 添加待办 → `/todo/create`

5. **最近待办（最多 3 条）**
   - 仅展示未完成待办，支持复选框快速完成

6. **下拉刷新**：刷新物品和待办数据

**异常状态**: 加载失败显示 Snackbar 错误提示；无待办时隐藏"最近待办"区域。

### 2.2 工作台 `/(tabs)/workbench`

**职责**: 全部功能入口分组展示，不承载业务数据、列表或表单。

布局:
- 页面标题"功能入口"
- 3 列入口网格，每个入口 = 彩色图标块 + 功能名称

| 分组 | 入口 | 路由 |
|---|---|---|
| 核心入口 | 物品管理 | `/item/list` |
| 核心入口 | 待办管理 | `/todo/list` |
| 核心入口 | 消息 | `/(tabs)/messages` |
| 管理工具 | 分类管理 | `/settings/category-manage` |
| 管理工具 | 位置管理 | `/settings/location-manage` |
| 管理工具 | 模板管理 | `/settings/templates` |
| 生活记录 | 借用管理 | `/settings/borrowings` |
| 生活记录 | 日历视图 | `/settings/calendar` |
| 数据与提醒 | 数据统计 | `/settings/stats` |
| 数据与提醒 | 通知中心 | `/settings/notifications` |
| 数据与提醒 | 数据管理 | `/settings/data` |
| 数据与提醒 | 资产总览 | `/settings/assets` |
| 数据与提醒 | 桌面快捷入口 | `/settings/widgets` |

**异常状态**: 工作台不依赖远程数据，无加载态；路由目标页负责各自的状态处理。

### 2.3 底部导航

| Tab | 图标 | 路由 | 规则 |
|---|---|---|---|
| 首页 | `home` | `/(tabs)` | 默认入口 |
| 工作台 | `grid` | `/(tabs)/workbench` | 功能入口 |
| 消息 | `message-text-outline` | `/(tabs)/messages` | 未读小红点 |
| 我的 | `account` | `/(tabs)/settings` | 个人资料 + 系统设置 |

当前选中橙色，未选中灰色；Tab Bar 黑色背景。

### 2.4 物品管理

#### 物品列表 `/item/list`

- 顶部自定义标题："物品管理" + 总数胶囊
- 右侧工具：批量选择、排序
- 搜索框：深色表面、主色聚焦边框、清除按钮
- 筛选器：分类/位置/状态 Picker 单选（默认全部），底部弹出选项列表
- 列表卡片：缩略图/图标、名称、描述、位置标签、分类标签
- 左滑删除 + 二次确认弹窗
- 下拉刷新
- FAB 进入 `/item/create`
- 批量模式：复选框 + 底部浮动批量栏（全选、已选数量、删除确认）

#### 物品创建 `/item/create` / 编辑 `/item/[id]`

- 必填：物品名称、分类（网格选择）、存放位置（网格选择）
- 选填：图片（最多 5 张）、描述、条码、到期日期、保质期提醒
- 价值字段：购买价格、购买日期、当前估值、货币、折旧率
- 编辑页附加：共享设置入口、借用记录入口
- 底部固定"取消 + 保存/保存修改"
- 创建和编辑复用同一页面，通过 URL 参数区分

### 2.5 待办管理

#### 待办列表 `/todo/list`

- 顶部自定义标题："待办管理" + 待完成数胶囊
- 分段筛选：全部 / 待完成 / 已完成
- 附加筛选：今日、逾期、有提醒
- 右侧工具：拖拽排序开关、排序
- 待办卡片：复选框、标题、描述、优先级标签、截止日期
- 点击复选框快速完成，点击卡片进入编辑页
- 左滑删除 + 二次确认弹窗
- 已完成项目删除线 + 灰色

#### 待办创建 `/todo/create` / 编辑 `/todo/[id]`

- 必填：标题
- 选填：分类、优先级（紧急/普通/低）、描述、图片、截止日期、提醒时间
- 提醒集成：设置提醒自动调度通知，修改时取消旧通知重建新通知，删除时取消关联通知
- 编辑页附加：完成状态切换、共享设置入口

### 2.6 分类管理 `/settings/category-manage`

- 列表：系统预设（只读）+ 用户自定义
- 系统预设不可删除，有视觉标识
- 分类显示：名称、类型（物品/待办）、颜色、图标
- 新增：名称、类型选择、颜色选择器、图标选择
- 左滑删除 + 二次确认（仅自定义分类可删）
- 与物品/待办表单的分类选择器联动

### 2.7 位置管理 `/settings/location-manage`

- 列表：系统预设（只读）+ 用户自定义（支持树形层级）
- 系统预设不可删除
- 新增：名称、可选父位置、自动设置层级
- 左滑删除 + 二次确认（仅自定义位置可删）
- 与物品表单的位置选择器联动

### 2.8 消息 `/(tabs)/messages`

**消息列表**
- 好友消息：仅已通过验证的好友
- 系统通知：好友申请、到期提醒、同步结果
- 未读小红点，下拉刷新
- 右上加号 → 添加好友 Sheet

**添加好友**
- 搜索用户（邮箱/用户名模糊匹配，300ms 防抖）
- 填写验证消息并发送
- 对方同意前不可建聊，申请结果通过系统通知反馈

**好友操作**
- 置顶/取消置顶
- 共享设置：物品/待办的 view/edit 权限
- 左滑删除 + 小弹窗确认；删除后清理共享和聊天入口

**聊天页 `/message/[id]`**
- 文字气泡、系统消息、物品/待办卡片消息
- 物品卡片：缩略图 + 名称 + 位置 + "打开"按钮
- 待办卡片：标题 + 优先级 + "标记完成"按钮
- 输入组件：文字输入 + 发送 + 快速分享物品/待办入口

### 2.9 模板管理 `/settings/templates`

- 分段：物品模板 / 待办模板
- 列表：名称、图标、类型、使用次数
- 新增：基于当前记录保存为模板
- 套用：选择模板 → 进入物品/待办创建页并预填字段
- 左滑删除 + 二次确认

### 2.10 借用管理 `/settings/borrowings`

- 列表：借用人、物品、借出/归还日期、状态标签（借出中/已归还/逾期）
- 从物品编辑页进入时保留当前物品上下文
- 新增：选择物品、借用人、联系方式、借出日期、预计归还日期
- 归还操作：点击确认归还
- 左滑删除 + 二次确认

### 2.11 日历视图 `/settings/calendar`

- 月视图网格，今天高亮
- 有待办或事件的日期有标记
- 点击日期显示当天的待办/提醒列表
- 提醒编辑沿用待办编辑页

### 2.12 数据统计 `/settings/stats`

- 待办完成率 + 进度条
- 下一步提示：优先逾期待办 → 今日待办 → 节奏正常
- 核心数字：待完成、已完成、近 7 天完成、物品总数
- 逾期待办列表（最多 4 条，按截止时间升序）
- 物品分类分布：横向占比条（最多 6 个分类）
- 不使用折线图/饼图/柱状图组合

### 2.13 通知中心 `/settings/notifications`

- 列表：待办提醒、物品统计、系统通知、推送通知
- 已读/未读：未读有橙色背景 + 边框高亮 + 标题加粗 + 红色圆点
- Tab 筛选：全部 / 未读 / 已读
- "全部标记为已读"按钮（未读 Tab 下）
- 点击未读通知自动标记已读并深链跳转
- 已读状态 AsyncStorage 持久化，重启后保持

### 2.14 数据管理 `/settings/data`

- 导出：JSON / CSV 格式
- 导入：预览、去重（按分类+名称/父级+位置复用）、关联重映射 → 导入报告（成功/跳过/失败）

### 2.15 资产总览 `/settings/assets`

- 总资产（所有物品 current_value 合计）
- 分类资产分布（各分类物品价值占比）
- 物品资产明细列表
- 最近价值变化记录（从 life_value_history 读取）

### 2.16 桌面快捷入口 `/settings/widgets`

- PWA 安装说明
- 统计摘要预览（物品数、待办数）
- 待办预览（今日待办、逾期待办）
- Web / 非 Web 降级说明

### 2.17 我的 `/(tabs)/settings`

- 个人信息卡片：头像、用户名、邮箱 → 点击进入 `/settings/account`
- 账号与安全：账号管理、修改密码、退出登录（小弹窗确认）
- 偏好设置：主题设置（系统/浅色/深色）、语言（中文/英文）
- 数据与支持：数据同步（手动同步）、数据管理、反馈建议、版本信息

### 2.18 全局搜索（首页搜索图标触发）

- 统一结果页/Sheet
- 按分类展示：功能入口、物品、待办、分类、位置
- 结果点击直达对应编辑页或管理页
- 搜索时自动补拉分类/位置数据

### 2.19 认证页

**登录 `/auth/login`**
- 邮箱 + 密码输入 + "登录"按钮
- "忘记密码" → 密码重置流程
- "没有账号？注册" → `/auth/register`
- 仅展示邮箱密码登录，不展示 OAuth 入口

**注册 `/auth/register`**
- 邮箱 + 密码 + "注册"按钮
- 注册成功自动登录

**密码重置 `/auth/reset-password`**
- 输入邮箱 → 发送重置邮件

## 3. 数据模型关系

```
用户 (auth.users)
 ├── 资料 (life_profiles) 1:1
 ├── 物品 (life_items) 1:N
 │   ├── 借用记录 (life_borrowings) 1:N
 │   ├── 价值历史 (life_value_history) 1:N
 │   ├── 分类 (life_categories) N:1
 │   └── 位置 (life_locations) N:1
 ├── 待办 (life_todos) 1:N
 │   └── 分类 (life_categories) N:1
 ├── 模板 (life_templates) 1:N
 ├── 反馈 (life_feedback) 1:N
 ├── 提醒日志 (life_reminder_logs) 1:N
 ├── 好友关系 (life_friendships) 1:N
 ├── 对话 (life_conversations) N:M
 │   └── 消息 (life_messages) 1:N
 └── 共享 (life_shares) 1:N (作为 owner 或 shared_with)
     ├── 指向物品 (life_items)
     └── 指向待办 (life_todos)
```

## 4. 交互规则

- 所有列表页支持左滑删除 + 应用内小弹窗确认
- 所有列表页支持下拉刷新
- 所有新增/编辑页底部固定"取消 + 保存/保存修改"
- 搜索默认是标题栏图标，只有主动搜索态才展示输入框
- 删除、退出、清理数据等危险操作统一使用应用内小弹窗，不调用原生 confirm/Alert
- 时间字段前端发送北京时间，后端入库转 UTC ISO，返回转北京时间+08:00
