## 1. 产品定位

<a id="1-product-positioning"></a>

LifeTracker 是一款**个人生活管理工具**，帮助用户追踪物品位置、管理待办事项，通过分类和位置体系实现生活条理化。

**核心价值主张**:
- 找东西不再翻箱倒柜——记录物品存放位置
- 待办不再遗漏——设置优先级和提醒
- 数据可视化——通过统计了解自己的生活模式

**竞品定位**: 介于备忘录类应用（Notion、备忘录）和物品管理类应用之间，轻量级、开箱即用、无需配置。

## 2. 用户画像

<a id="2-user-persona"></a>

### 主要用户群体
- **个人用户**，需要管理日常物品和任务
- **生活爱好者**，喜欢记录和整理
- **忙碌上班族**，需要快速记录待办和物品位置

### 使用场景
| 场景 | 描述 |
|------|------|
| 找东西 | 忘记某件物品放在哪里，通过搜索快速定位 |
| 物品整理 | 搬家或整理房间后，更新物品位置 |
| 任务管理 | 创建待办事项，设置优先级和截止日期 |
| 数据统计 | 回顾物品和待办的使用情况 |

## 4. 功能详细需求

<a id="4-feature-details"></a>

### 4.1 用户认证

#### 4.1.1 邮箱密码登录
- 用户使用邮箱和密码登录
- 登录成功进入首页，失败显示错误提示
- 账号不存在/密码错误时给出明确提示
- 登录态持久化，下次打开应用自动登录

#### 4.1.2 邮箱密码注册
- 新用户通过邮箱和密码注册
- 注册成功自动登录，失败显示错误提示
- 邮箱已注册时提示

#### 4.1.3 密码重置
- 输入邮箱 → 发送重置邮件 → 用户点击邮件链接重置密码

#### 4.1.4 第三方登录（可选增强）
- **当前版本**: 仅开放邮箱密码登录，登录页不展示微信/Google 等入口
- **后续条件**: OAuth provider 配置、回调链路和线上验收完成后作为稳定登录方式

#### 4.1.5 登出
- 使用应用内小弹窗二次确认
- 清除本地 token，跳转到登录页

#### 4.1.6 路由守卫
- 未登录自动跳转到登录页
- 已登录访问认证页面自动跳转到首页

---

### 4.2 首页

#### 4.2.1 通知入口
- 右上角铃铛图标，有新通知时显示红点 + 抖动动画
- 点击进入通知中心 `/settings/notifications`

#### 4.2.2 "今天要看"行动区
- 展示逾期待办、今日待办、即将过期物品
- 每项可点击跳转到对应编辑页
- 无待办/无逾期时不展示该区域

#### 4.2.3 数据统计卡片
- **物品**: 显示物品总数，点击跳转到 `/item/list`
- **待办**: 显示待完成数量，点击跳转到 `/todo/list`
- **完成**: 显示已完成数量，不可点击

#### 4.2.4 快捷操作
- **添加物品**: 直接进入创建物品页面 `/item/create`
- **添加待办**: 直接进入创建待办页面 `/todo/create`
- 固定 2 个主按钮，不新增低频入口

#### 4.2.5 下拉刷新
- 支持下拉刷新物品和待办数据

---

### 4.3 物品管理

#### 4.3.1 物品列表
- 显示所有物品，每张卡片显示: 分类图标、名称、描述、位置标签、分类标签
- 点击列表项进入物品编辑页；被共享且无编辑权限时展示只读摘要 Sheet
- 左滑显示删除按钮 + 二次确认弹窗
- 下拉刷新

#### 4.3.2 搜索
- 支持搜索物品名称、描述、分类、位置、条码
- 300ms 防抖延迟，聚焦时搜索框高亮（主色边框）
- 有内容时显示清除按钮

#### 4.3.3 分类筛选
- Picker 单选，默认"全部"
- 选项包含系统预设 + 用户自定义物品分类
- 选中后触发按钮以主色高亮

#### 4.3.4 位置与状态筛选
- 位置 Picker 单选，默认"全部"
- 状态 Picker 单选，默认"全部"，选项：已过期、7天内到期、有价值、未估值

#### 4.3.5 排序
- 底部弹出选择器：添加时间（默认，最新在前）、名称（A-Z）、分类

#### 4.3.6 批量模式
- 点击多选图标进入批量模式
- 显示批量操作栏（全选、已选数量、删除 + 二次确认）

#### 4.3.7 创建/编辑物品
- **必填字段**: 物品名称、分类（网格选择）、存放位置（网格选择）
- **选填字段**: 图片（最多 5 张）、描述、条码、到期日期、保质期提醒
- **价值字段**: 购买价格、购买日期、当前估值、货币、折旧率
- **表单验证**: 名称为空提示、未选分类/位置提示，实时清除错误提示
- **智能录入建议（实验）**: 图片区的模拟建议（名称/分类预填候选），需人工确认
- 创建和编辑复用同一页面，通过 URL 参数区分

#### 4.3.8 分类/位置选择
- 表单内选择支持搜索和最近使用
- 与管理页数据一致

#### 4.3.9 图片处理
- 选择: 拍照或从相册选择
- 压缩: 原生平台压缩至 1024px 宽，质量 0.8；Web 端跳过压缩
- 上传: 调用后端 API 上传到 Supabase Storage
- 预览: 全屏查看，支持滑动切换
- 删除: 点击图片预览中的删除按钮移除

---

### 4.4 待办管理

#### 4.4.1 待办列表
- 每张卡片显示: 复选框、标题、描述、优先级标签、截止日期
- 左滑显示删除按钮 + 二次确认弹窗
- 点击复选框快速切换完成状态
- 点击卡片进入待办编辑页；被共享且无编辑权限时展示只读摘要 Sheet
- 已完成项目标题显示删除线 + 灰色

#### 4.4.2 筛选 Tab
- **全部**: 显示所有待办
- **待完成**: 仅显示未完成的待办
- **已完成**: 仅显示已完成的待办
- 附加筛选: 今日、逾期、有提醒

#### 4.4.3 排序
- 底部弹出选择器：添加时间（默认）、优先级（高到低）、名称（A-Z）

#### 4.4.4 创建/编辑待办
- **必填字段**: 标题
- **选填字段**: 分类、优先级选择（紧急/普通/低）、描述、图片（最多 5 张）、截止日期、提醒时间
- **提醒集成**: 设置提醒时间后自动调度通知；修改时取消旧通知、设置新通知；删除待办时取消关联通知

#### 4.4.5 优先级系统
| 级别 | 标签 | 颜色 | 图标 |
|------|------|------|------|
| 3 | 紧急 | 红色 (#EF4444) | alert-circle |
| 2 | 普通 | 橙色 (#F59E0B) | alert |
| 1 | 低 | 绿色 (#10B981) | check-circle |

---

### 4.5 分类管理

- 显示物品分类和待办分类，包含系统预设 + 用户自定义
- 系统预设不可删除，有视觉标识
- 每个分类显示名称、类型、颜色、图标
- 创建分类: 名称、类型（物品/待办）、颜色选择器、图标选择
- 删除: 仅可删除用户自定义分类，需确认；删除后关联字段变为空

---

### 4.6 位置管理

- 显示位置列表（支持层级），包含系统预设 + 用户自定义
- 系统预设不可删除
- 创建位置: 名称、可选父位置、自动设置层级
- 删除: 仅可删除用户自定义位置，需确认；删除后关联字段变为空

---

### 4.7 数据统计

- **待办完成率**: 已完成 / 总待办 × 100%，配合进度条
- **下一步提示**: 优先逾期待办，其次今日待办；无风险时展示"节奏正常"
- **核心数字**: 待完成、已完成、近 7 天完成、物品总数
- **逾期待办**: 未完成且截止时间已过，按截止时间升序排列，最多 4 条
- **物品分类分布**: 按分类展示数量和占比，最多 6 个分类，横向占比条

---

### 4.8 通知中心

- **待办提醒**: 提醒时间到达时自动添加
- **已读/未读状态**: 未读有橙色背景 + 边框高亮，标题加粗，右侧红色圆点
- **Tab 筛选**: 全部、未读、已读 + "全部标记为已读"
- **首页铃铛动画**: 有未读时抖动 + 红点
- **实时推送（Socket）**: 后端 `ReminderScheduler` 每分钟检查，防重复机制，Socket 事件推送
- **数据持久化**: 已读状态通过 AsyncStorage 本地持久化

---

### 4.9 账号管理

- 头像（支持自定义上传）、用户名、邮箱
- 编辑资料: 用户名、邮箱、头像
- 修改密码: 当前密码 + 新密码 + 确认密码
- 退出登录: 应用内小弹窗确认

---

### 4.10 反馈建议

- 反馈内容（必填，多行文本）
- 联系方式（选填）
- 数据存入 `life_feedback` 表

---

### 4.11 消息模块

#### 4.11.1 消息列表
- 好友消息：仅展示已通过好友验证的用户
- 系统通知：好友申请、到期提醒、同步结果等
- 未读状态使用小红点
- 下拉刷新，右上角加号打开"添加好友"页

#### 4.11.2 添加好友与验证申请
- 搜索用户: 输入邮箱或用户名，模糊匹配
- 发送申请: 填写验证消息并发送
- 建聊规则: 对方同意前不得建立聊天会话；申请结果通过系统通知反馈

#### 4.11.3 好友操作
- 置顶 / 取消置顶
- 共享设置: 对单个好友设置物品/待办的查看或编辑权限
- 删除好友: 左滑触发 + 小弹窗确认；删除后清理共享权限和聊天入口

#### 4.11.4 聊天页
- 物品卡片消息: 缩略图 + 名称 + 位置 + "打开"按钮
- 待办卡片消息: 标题 + 优先级 + "标记完成"按钮
- 文字消息: 气泡样式，区分发送者/接收者
- 系统通知消息: 灰色背景
- 输入组件: 文字输入 + 发送 + 分享物品/待办入口

#### 4.11.5 分享联动
- 物品/待办编辑页 → 选择已通过好友 → 创建 share 记录 + 写入权限 + 对话卡片消息
- 分享收口到好友上下文，不做独立共享管理入口

<a id="5-product-roadmap"></a>

### 5.1 v1.4.4 发布前真实验收（当前阶段）

v1.4.4 不新增业务入口，目标是把现有主干增强转成可交付版本。

| 主线 | 范围 | 状态 |

## 6. 交互规范

<a id="6-interaction-specs"></a>

### 6.1 页面导航
- Tab 导航: 底部 4 个 Tab（首页、工作台、消息、我的）
- 工作台: 功能卡片矩阵（3 列网格），点击跳转二级功能页面
- Stack 导航: 列表、新增、编辑页作为子页面推入导航栈
- 返回按钮: 新增/编辑/操作页标题左侧返回图标
- 手势返回: 支持侧滑返回（原生平台）

### 6.2 操作反馈
- **成功**: Toast 提示，绿色，1.5 秒后自动消失
- **错误**: 页面内错误提示、Toast 或小弹窗
- **加载**: Loading 遮罩或骨架屏
- **确认**: 应用内小弹窗（删除、退出、清理数据、删除好友等敏感操作）

### 6.3 手势交互
- 左滑删除: 所有可删除的列表项
- 下拉刷新: 所有列表页
- 侧滑返回: 子页面
- 点击复选框: 快速切换待办完成状态
- 拖拽排序: 待办列表支持长按拖拽

### 6.4 搜索组件
- 搜索默认展示为标题栏右侧线性图标
- 点击后进入搜索态/搜索页/搜索 Sheet，只有主动搜索态才展示输入框

### 6.5 空状态
- 所有列表页必须有空状态引导（图标 + 标题 + 描述 + 操作按钮）

### 6.6 表单交互
- 必填字段标记红色星号，实时表单验证
- 新增/编辑页标题左侧返回图标，底部固定"取消 + 保存/保存修改"
- 删除统一从列表项左滑露出，不放在编辑页内

### 6.7 加载状态
- 首次加载: 骨架屏（3 条）
- 刷新: 下拉刷新指示器
- 提交: 按钮 loading 状态

## 8. 技术约束

<a id="8-technical-constraints"></a>

### 8.1 平台适配
- 所有功能必须同时适配 **Web (PWA)** 和 **Android**
- 不能使用仅单端可用的方案
- Web 端不支持: `expo-image-manipulator`（压缩）、`expo-notifications`（推送）
- Web 端需使用替代方案

### 8.2 数据库约束
- 所有表使用 `life_` 前缀，开发和生产共用数据库，所有表禁用 RLS（后端使用 service_role key + 应用层守卫鉴权）
- 分类/位置的 `user_id` 为 NULL 表示系统预设
- 后端表名使用复数，前端类型名使用单数

### 8.3 认证约束
- 认证操作使用 anon key，service_role key 仅用于管理员操作
- Token 存储在本地（AsyncStorage / localStorage）

### 8.4 API 约束
- 所有接口（除登录/注册）需 Bearer Token 认证
- user_id 由后端从 token 自动注入
- 错误统一格式: `{ message: string }`

### 8.5 文件上传约束
- 存储桶: `items-images` (public)，路径格式: `{user_id}/{timestamp}_{filename}`

### 8.6 性能约束
- 列表数据考虑分页和缓存，搜索使用 300ms 防抖
- 图片上传前压缩（原生平台）
