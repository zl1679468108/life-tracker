- `EventsGateway.emitMessageCreated()` 往 `conversation:{id}` 房间 emit，而前端 socket 只 join 了 `user:{userId}` 房间

**Bug 2：聊天页面的 cleanup 调用了 `removeAllListeners()` 清空所有 store 级监听器**
- 2 个页面（消息列表页、聊天详情页）的 cleanup 都调用了 `removeAllListeners()`
- 这移除了 itemStore/todoStore/notificationStore 等在模块级别注册的 socket 监听器
- 修复：新增 `off*` 方法族，cleanup 改用只移除本页面的监听器

## 16:22 — 修复 Select 筛选器文字不显示

问题：用户截图显示三个筛选 pill 只显示下拉箭头，默认文字"全部分类/全部位置/全部状态"不可见。

根因：`frontend/components/ui/Select.tsx` 的 `triggerText` 同时设置了 `flex: 0` 与 `numberOfLines={1}`，在 web 端 flex 布局下文字容器被压缩到 0 宽度，导致文字不渲染。

1. 将文字用 `View` 包裹并设置 `flexShrink: 1`，移除 `numberOfLines={1}`
2. 给 `triggerText` 加固定 `lineHeight: 18`，确保 web 行高正确
